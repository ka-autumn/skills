---
name: handoff
description: 現在の会話を圧縮し、別のエージェントが引き継げる引き継ぎドキュメントにまとめる。
argument-hint: "次のセッションでは何をしますか？"
---

新しいエージェントが作業を継続できるよう、現在の会話を要約した引き継ぎドキュメントを作成してください。保存先は、現在のワークスペースではなく、ユーザーの
OS の一時ディレクトリにしてください。

ドキュメントには「suggested skills」セクションを含め、エージェントが起動すべきスキルを提案してください。

他の成果物（PRD、計画、ADR、Issue、コミット、コードの差分）に既に記録されている内容は重複させないこと。代わりにパスや URL
で参照すること。

API キー、パスワード、個人を特定できる情報などの機微情報は伏せること。

ユーザーが引数を渡した場合は、それを次のセッションで重点的に扱う内容の説明とみなし、それに合わせてドキュメントを調整してください。
